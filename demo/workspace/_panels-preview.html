Partie 1 — Implémentés : fragments déjà câblés dans le pipeline section-pipeline.ts et convertis automatiquement lors du rendu markdown.

Partie 2 — Proposés (génériques) : 10 panneaux d'intérêt éditorial général (pros/cons, checklist, FAQ…).

Partie 3 — AO / RFP / RFQ : 11 panneaux taillés pour les réponses à marchés publics (exigence CCTP, référence similaire, SLA, tranche ferme, hypothèse, annexe, registre des risques…).

Partie 4 — IT / TMA / projets informatiques : 18 panneaux pour les mémoires techniques dédiés aux projets SI (stack, environnements, tickets, incidents, tests, monitoring, release, sprint, RACI, migration, roadmap, accessibilité, sécurité, user-story, cycles de vie évolution/anomalie, org-chart…).

Partie 5 — Compléments : 23 panneaux couvrant gouvernance (ADR, escalade, rétrospective, engagement), commercial (scope, feature-matrix, BPU, échéancier), produit (persona, journey, data-flow), exploitation (runbook, changelog, on-call, capacity), conformité (RGPD, audit-trail), visuels chiffrés (progress, gauge, comparison) et narration (executive-summary, stakeholders, version-history).

Le HTML des panneaux proposés (parties 2 à 5) est fidèle à ce que produirait le pipeline une fois ces containers reconnus ; les CSS correspondantes vivent pour l'instant dans cette page uniquement (non injectées dans pdf.css).
