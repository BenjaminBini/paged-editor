Differences: preview vs. PDF export

Aspect Preview render PDF export
Heading IDs h-{gen}-{slug} (per-render offset) h-{counter}-{slug} (continuous across files)
gen value currentGen (N) — stale render detection 0 — not used
Logo format Blob URL (fast, smaller HTML) Base64 data-URI (self-contained for PDF)
Coverage Active tab only All .md files in workspace
Headings collector Not used Accumulates across all sections → TOC
Web output iframe in preview-wrapper window.open() → user prints with Cmd+P
Electron output iframe in preview-wrapper Offscreen BrowserWindow → printToPDF → pdf-lib → viewer iframe

platform.js abstracts over two backends — modules never call window.electronAPI directly

Operations and routing

Operation Electron IPC Web REST
Read file invoke("read-file", path) GET /api/files/:name
Write file invoke("write-file", path, content) PUT /api/files/:name {content}
Delete file invoke("delete-file", path) DELETE /api/files/:name
List files invoke("read-dir", path) GET /api/files
File mod time invoke("get-file-mod-time", path) GET /api/files/:name/meta
App state read invoke("get-app-state") → app-state.json GET /api/config + localStorage
App state write invoke("set-app-state", partial) localStorage.setItem() only
Open dialog invoke("show-open-file-dialog") Not available
PDF export invoke("preview-pdf", html) → BrowserWindow window.open(blobUrl) → user prints
