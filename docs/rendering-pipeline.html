Rendering Pipeline

From Markdown source to paginated A4 preview — in-DOM Paged.js mode

1
Trigger
User types in CodeMirror → editor.on("input") fires → scheduleRender(800) debounce → triggerRender().
Render requests are queued: a generation counter is incremented on each new request. Stale in-flight renders are discarded when a newer generation overtakes them. Only one render runs at a time (isRendering gate).
render.js scheduleRender() triggerRender() flushRenderQueue()
2
Tab Dispatch
The active tab kind determines which HTML builder is called:
Cover tab → buildCoverRenderResult()
TOC tab → buildTocRenderResult()
Markdown tab → renderMarkdown()
All three return a RenderResult object: { sectionHtml, headerText, rootPageName, sourceBlocks, lineStarts, lineNumberOffset }
render.js memoire-views.js buildCoverRenderResult() buildTocRenderResult() render-pipeline.js renderMarkdown()
3
Markdown → HTML (markdown tabs)
Markdown is parsed by marked with custom renderers and extensions. Frontmatter is extracted first. Extensions handle heading numbering, French typography, section colors, underlines, and Mermaid diagrams. Source-line tracking metadata (sourceBlocks) is collected for the line map.
Raw MD → Frontmatter → marked parse → French typo → Heading nums → sectionHtml
render-pipeline.js renderMarkdown() marked extensions
4
Preview Document Assembly
buildPreviewDocument() wraps sectionHtml in a .pdf-preview-root div (not a full HTML document — Paged.js receives raw HTML markup). Running header and gradient elements are injected here if not a special page type. The resulting HTML is sanitized by DOMPurify (allows blob: and file: URIs for Electron compatibility).

Normal section

Includes .pdf-page-gradient and .pdf-running-header running elements consumed by Paged.js @top margin rules.

Cover / TOC pages

Use a rootPageName (e.g. "sommaire") which applies a named @page rule. Running elements and @page :first overrides are suppressed.

preview-renderer.js buildPreviewDocument() sanitizePreviewHtml() DOMPurify
5
In-DOM Paged.js Pagination
The previous Paged.js instance is destroyed (dispose() tears down chunker + polisher). previewPages.replaceChildren() clears the DOM. Then previewer.preview(html, styles, container) runs Paged.js directly in the live DOM — no iframe, no blob URL, no postMessage.
Styles (pdf.css, paged.css, fonts, logo CSS) are passed as an array of named { "name.css": cssText } objects so Paged.js can reference them by name in @import rules.
dispose() → replaceChildren() → previewer.preview(html, styles, previewPages) → .pagedjs_page nodes in DOM
preview-renderer.js PreviewRenderer.render() buildPreviewStyles() paged.polyfill.js
6
Scale
After previewer.preview() resolves, scaleSurface() measures the rendered .pagedjs_page width and applies transform: scale() to fit the preview pane, factoring in the user zoom level. The wrapper div is resized to match the scaled content so the scrollbar reflects true document height.
render.js scaleSurface() measurePreviewSurface()
7
Line Map & Scroll Sync
rebuildLineMap() is called after scaleSurface() so getBoundingClientRect() reflects the final scaled layout. It maps editor source line numbers to preview Y positions by reading data-source-line attributes injected during Markdown parsing. Bidirectional scroll sync and click-to-navigate use this map.
rebuildLineMap() sourceBlocks → Y positions emit("section-ready")
preview-renderer.js rebuildLineMap() line-map.js buildLineMap() sync.js
