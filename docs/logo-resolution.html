Logo Resolution

From project.json logo entry to URL embedded in cover and preview HTML

1
Project Data Normalization
When project.json is loaded (or created for the first time), normalizeProjectData() in memoire-views.js applies defaults to each logo entry via normalizeLogoEntry(entry, key).

Defaults are driven by the LOGO_DEFAULTS map: candidat defaults to the built-in BEORN logo; partenaire defaults to the built-in Liferay logo (both with showInCover: true). acheteur has no default file. 
// LOGO_DEFAULTS in memoire-views.js
{
  candidat:   { file: "assets/beorn-logo.png",    showInCover: true  },
  partenaire: { file: "assets/liferay-logo.svg",  showInCover: true  },
}

// Resulting entry when project.json has no logos.candidat:
{
  file:         "assets/beorn-logo.png",
  showInCover:  true,
  coverWidth:   180,
  showInFooter: false,
  ...
}
memoire-views.js normalizeLogoEntry() LOGO_DEFAULTS
2
Built-in Logo Assets
All built-in logos are static files inside packages/base/assets/. They are served at a stable relative path — assets/<name> — by both runtimes:

Electron

The <base href> points at the editor package root. assets/beorn-logo.png resolves to file:///…/packages/base/assets/beorn-logo.png.

Web (Express)

express.static("packages/base") serves the editor root. GET /assets/beorn-logo.png is served from disk.

No preloading, no blob URLs, no data URIs. The file field in project.json is used verbatim as an <img src> or CSS url() value. To add a new built-in logo, drop the file in assets/ and add an entry to CARD_PRESETS (cover form) and LOGO_DEFAULTS (memoire-views).
assets/beorn-logo.png assets/liferay-logo.svg assets/lumapps-logo.svg
3
Cover Form — Preset Toggle & Upload
The Cover form builds a logo card for each key (candidat, partenaire, acheteur). Cards for keys that have entries in CARD_PRESETS render a toggle row of preset buttons plus an Upload... button. acheteur has no presets and gets a plain Choose / Remove UI instead. 
// CARD_PRESETS in cover-form.js
{
  candidat:   [ { label: "BEORN",   file: "assets/beorn-logo.png"   } ],
  partenaire: [
    { label: "Liferay", file: "assets/liferay-logo.svg" },
    { label: "LumApps", file: "assets/lumapps-logo.svg" },
  ],
}
Preset btn click → r.file = preset.file
r._dataUrl = null → syncSourceToggle() + updateLogoPreview()
Upload... click → fileInput.click() → FileReader → writeBinaryFile → r.file = destPath
syncSourceToggle(key) iterates the card's .logo-source-btn elements and applies the active class to whichever button's data-preset-file matches r.file (when no upload is pending). The Upload button activates when r._dataUrl is set or the file doesn't match any preset.
cover-form.js CARD_PRESETS buildLogoCard() syncSourceToggle() updateLogoPreview()
4
User File Upload
When the user picks a file via Upload...:
• The destination path is prefixed with the logo key to avoid collisions: "assets/" + key + "-" + file.name (e.g. assets/partenaire-acme.png)
• FileReader.readAsDataURL(file) encodes the file as base64
• writeBinaryFile(destPath, base64) writes it to the workspace assets/ folder
• r.file is set to destPath; r._dataUrl holds the data URL for the in-form thumbnail
Pick file → FileReader → base64 → writeBinaryFile → workspace assets/
→ → r._dataUrl → thumbnail preview + r.file = "assets/key-name.ext"
Upload happens at file-selection time, before the project is saved. This ensures the file is on disk when the JSON is written and the preview renders. The key prefix (candidat-, partenaire-) prevents two fields from overwriting each other when they upload files with the same name.
cover-form.js fileInput change handler platform.js writeBinaryFile() → /api/assets (web) or IPC (Electron)
5
Cover Form Thumbnail
updateLogoPreview(key) resolves the URL to display in the form's logo thumbnail:
r._dataUrl set → use data URL directly (freshly uploaded file)
r._dataUrl null → use r.file as-is → relative URL → browser resolves via <base href>
Because built-in assets are served at the same relative path as workspace files, r.file = "assets/beorn-logo.png" resolves correctly in both cases with no special handling. The thumbnail appears immediately on form load.

6
Cover & TOC HTML Rendering
buildCoverHtml(fm) and buildSommaireHtml(headings, fm) in document.js read fm.logos.candidat and fm.logos.partenaire. When showInCover is true and a file path is set, the path is embedded directly as the src of an <img> element — no transformation.

Live preview & PDF export

Both use the same HTML. The preview renderer and the export wrapper both have <base href> pointing at the editor root, so assets/beorn-logo.png resolves correctly. PDF export is not self-contained — it requires network/file access at print time.

Adding a new built-in logo

Drop the file in packages/base/assets/. Add an entry to CARD_PRESETS in cover-form.js and optionally to LOGO_DEFAULTS in memoire-views.js. No fetch, no preload, no blob URL needed.

fm.logos.candidat.file → <img src="assets/beorn-logo.png"> → resolved by <base href>
